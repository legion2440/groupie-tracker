# Groupie tracker - Visualizations

Визуальная часть Groupie Tracker объединяет адаптивную музыкальную стилистику, светлую и тёмную темы, интерактивные SVG-элементы и понятную обратную связь. Интерфейс построен без frontend-фреймворков, GIF, video и canvas.

## 🎨 Визуальная система

- единая палитра на CSS custom properties;
- Inter используется для основного текста, Montserrat — для заголовков;
- шрифты хранятся локально в WOFF2 и не требуют подключения внешнего CDN;
- карточки, фильтры, top-bar, footer и страница артиста используют общие дизайн-токены;
- акцентный фиолетовый цвет связывает интерактивные состояния, карту и музыкальные элементы;
- цветовые состояния дополняются формой, обводкой, положением и текстом, а не передаются только цветом.

В проекте нет отдельной библиотеки компонентов: интерфейс собран из семантического HTML, CSS и небольших независимых JavaScript-модулей.

## 🌙 Светлая и тёмная темы

- начальная тема учитывает `prefers-color-scheme` операционной системы;
- выбранная пользователем тема сохраняется в `localStorage`;
- небольшой inline-скрипт применяет класс темы до загрузки CSS и предотвращает FOUC;
- обе темы используют один набор семантических переменных: `surface`, `on-surface`, `primary`, `outline`, `error` и другие;
- SVG-иконки наследуют `currentColor` и автоматически адаптируются к теме.

## 💿 Карточки и виниловые анимации

Карточка артиста оформлена как виниловая пластинка в конверте:

- фотография обрезается по кругу;
- поверх изображения рисуются дорожки пластинки;
- центральное отверстие и тени усиливают эффект винила;
- при наведении конверт приподнимается, а пластинка вращается;
- имя артиста меняет цвет, сохраняя читаемость в обеих темах.

На странице артиста используется увеличенная версия пластинки. Она вращается медленнее, чтобы не отвлекать от основной информации.

Технически вращение выполняется CSS-анимацией `transform: rotate(...)`. Отдельные графические элементы интерфейса выполнены в SVG; GIF, video и canvas не используются.

## 🗺️ Tour Map

Страница артиста связывает список концертов со схематичной картой мира:

- фон континентов создаётся SVG-маской `world-map.svg`;
- каждая карточка концерта связана с точкой через `data-location-index`;
- выбор карточки активирует соответствующий marker;
- между предыдущей и новой точкой строится SVG-кривая Безье;
- SVG-самолёт движется по длине path и поворачивается по касательной;
- скорость полёта зависит от длины маршрута и ограничена диапазоном `1100–2000 ms`;
- маршрут использует отдельное easing для плавной посадки;
- активная точка пульсирует, а пунктир маршрута движется во время полёта;
- `ResizeObserver` пересчитывает геометрию после изменения размеров viewport.

> Точки карты сейчас являются визуальными позициями и не представляют реальные географические координаты концертных городов.

## 🎞️ SVG и анимации

Векторная графика используется для:

- favicon-пластинки;
- карты мира;
- маршрута перелёта;
- самолёта;
- иконок поиска, обновления, темы, фильтров и навигации.

### Анимированный favicon

`favicon.svg` содержит статическую SVG-пластинку, а `favicon.js` создаёт анимацию:

1. Загружает исходный SVG.
2. Находит группу `vinyl`.
3. Устанавливает новый угол `transform="rotate(...)"`.
4. Сериализует SVG в `data:image/svg+xml`.
5. Обновляет `href` favicon каждые `125 ms`.

Полный оборот занимает 3 секунды. Если SVG не загрузился или браузер запросил `prefers-reduced-motion: reduce`, остаётся статическая иконка.

### Reduced motion

Для пользователей с уменьшенной анимацией:

- favicon не вращается;
- самолёт сразу перемещается в конечную точку;
- маршрут и активное состояние остаются понятными без движения.

## 🎵 Deezer API track preview

При наведении на карточку артиста или большую пластинку запускается короткое превью трека из Deezer.

> Если браузер блокирует звук до первого пользовательского действия, нужно один раз кликнуть в любом месте страницы. Затем превью будет запускаться при наведении.

Как это работает:

- frontend запрашивает `/api/deezer-preview?artist=...` только при первом наведении;
- backend обращается к `https://api.deezer.com/search`;
- выбирается первый результат с непустым полем `preview`;
- серверный кеш хранит найденный URL или отсутствие результата;
- browser-side `Map` исключает повторные запросы в текущей вкладке;
- одновременно воспроизводится только один трек;
- при уходе курсора аудио останавливается и возвращается к началу;
- request token защищает от запуска устаревшего ответа после быстрого перехода между карточками;
- Deezer имеет таймаут 15 секунды;
- ошибка API, отсутствие трека или запрет autoplay не ломают страницу.

## 🧭 Навигация и состояния

- карточки ведут на человекочитаемые URL вида `/<artist-slug>`;
- legacy URL `/artist/{id}` получает постоянный redirect `301`;
- поиск и фильтры меняют только контейнер результатов;
- URL обновляется через History API;
- Back / Forward восстанавливают результаты и состояние элементов управления;
- при отсутствии совпадений показывается отдельный empty state с кнопкой Reset;
- кнопка обновления имеет состояния выполнения, успеха и ошибки;
- hover, focus, active и selected оформлены последовательно во всём интерфейсе.

## ♿ Доступность

- поисковая строка реализует ARIA combobox / listbox pattern;
- подсказки доступны стрелками, `Enter`, `Escape` и `Tab`;
- статус подсказок передаётся через `aria-live`;
- карточки концертных локаций доступны через `Enter` и `Space`;
- focus-visible состояния не зависят от hover;
- декоративные SVG скрыты от accessibility tree;
- range-контролы имеют связанные labels и outputs;
- поддерживается `prefers-reduced-motion`.

---

## 📱 Адаптивность

- сетка артистов перестраивается под ширину viewport;
- sidebar имеет стабильную ширину `288 px` на desktop и переходит в одноколоночную компоновку на узких экранах;
- карточки используют `aspect-ratio` и не искажают изображения;
- top-bar, footer, блок артиста и Tour Map имеют отдельные tablet/mobile правила;
- длинные имена артистов и локаций переносятся без выхода за контейнер;
- геометрия маршрута пересчитывается при изменении размера карты.

---

## ⚠️ HTTP-errors

- ошибки `400`, `404` и `500` отображаются через общий шаблон `error.html`;
- неизвестный маршрут не получает сырой ответ стандартной библиотеки Go;
- отсутствующий файл во встроенной статике также обрабатывается фирменной страницей `404`;
- тема и анимированный favicon подключены и на страницах ошибок.
