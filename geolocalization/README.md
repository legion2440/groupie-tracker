# Groupie tracker - Geolocalization

Geolocalization добавляет реальные географические координаты концертным локациям Groupie Tracker и отображает их на интерактивной Tour Map без сторонней картографической библиотеки.

## 📝 Назначение

Groupie Trackers API возвращает концертные локации строками вида:

```text
london-uk
belo_horizonte-brazil
abu_dhabi-united_arab_emirates
```

Geolocalization преобразует каждую уникальную локацию в:

```text
normalized key → latitude, longitude
```

Одна координата используется всеми артистами с той же локацией. Координаты не привязаны к карточке артиста и не дублируют данные Groupie Trackers API.

---

## 🔄 Поток данных

```text
Groupie Trackers /relation
        ↓
уникальные нормализованные location keys
        ↓
data/geocoding-cache.json
        ↓
Nominatim только для отсутствующих ключей
        ↓
проверка полного покрытия
        ↓
latitude / longitude в SSR-шаблоне
        ↓
Tour Map в браузере
```

При обновлении основного кеша приложение:

1. Параллельно загружает данные Groupie Trackers API.
2. Собирает уникальные локации из relations.
3. Проверяет их наличие в локальном JSON-справочнике.
4. Геокодирует только неизвестные локации.
5. Не публикует новый снимок данных, если координаты покрыты не полностью.

После заполнения JSON обычный запуск не зависит от доступности Nominatim.

## 💾 Постоянный справочник координат

Файл:

```text
data/geocoding-cache.json
```

Формат:

```json
{
  "version": 1,
  "locations": [
    {
      "key": "london-uk",
      "location": "London, UK",
      "latitude": 51.5074456,
      "longitude": -0.1277653,
      "source": "nominatim",
      "match": "exact"
    }
  ]
}
```

Особенности:

- ключи нормализуются и не зависят от регистра;
- записи сортируются по `key`;
- координаты проверяются по диапазонам latitude и longitude;
- JSON сериализуется детерминированно;
- запись выполняется через временный файл и замену целевого файла;
- повреждённый JSON и неподдерживаемая версия считаются ошибкой;
- путь можно переопределить переменной `GEOCODING_CACHE_PATH`.

Текущий справочник содержит:

```text
Unique locations: 189
Exact matches:    179
Fuzzy matches:     10
Manual entries:     0
Coverage:         100%
```

Эти числа относятся к текущему снимку Groupie Trackers API.

## 🌍 Геокодирование через Nominatim

Используется публичный OpenStreetMap Nominatim Search API.

### Exact-first matching

Для неизвестной локации сначала выполняется обычный запрос по полному сочетанию города и страны.

Результат принимается только после проверки:

- нормализованного названия;
- страны или country code;
- известных country aliases;
- регистра, разделителей и диакритики.

### Fuzzy fallback

Bounded fuzzy fallback выполняется только если обычный поиск:

- вернул пустой результат; или
- не дал допустимого exact match.

Правила fallback:

- выполняется не более одной дополнительной попытки;
- страна остаётся жёстким ограничением;
- используется общий bounded Levenshtein helper проекта;
- слабые и неоднозначные совпадения отклоняются;
- транспортные и серверные ошибки не маскируются fuzzy-поиском.

### Ограничения клиента

- запросы выполняются последовательно;
- минимальный интервал между запросами - 1 секунда;
- используется идентифицирующий `User-Agent`;
- каждый HTTP-запрос имеет timeout;
- успешные совпадения, fallback, отказы и ошибки логируются;
- API-ключ не требуется.

## 🗺️ Tour Map

Карта остаётся собственной SVG-визуализацией проекта.

- реальные координаты передаются через `data-latitude` и `data-longitude`;
- используется equirectangular projection в системе `1000 × 500`;
- markers сохраняют постоянный размер в экранных пикселях;
- SVG-маршрут строится между спроецированными точками;
- самолёт движется по path и поворачивается по касательной;
- камера автоматически подстраивается под общий охват тура;
- переход через `180° / -180°` обрабатывается как antimeridian;
- при resize карта, markers, route и plane пересчитываются;
- выбор концертной карточки не изменяет framing камеры;
- `prefers-reduced-motion` отключает полёт, но сохраняет маршрут и active state.

Карта не использует Leaflet, Mapbox, Google Maps, canvas или сторонний frontend-фреймворк.

## 🛠️ CLI

Для проверки и заполнения справочника используется отдельная команда:

```bash
go run ./cmd/geocode-locations
```

Поддерживаемые параметры:

```text
-cache      путь к JSON-справочнику
-base-url   endpoint Nominatim
-timeout    timeout одного HTTP-запроса
-interval   минимальный интервал между запросами
```

Пример:

```bash
go run ./cmd/geocode-locations   -cache data/geocoding-cache.json   -timeout 8s   -interval 1s
```

Команда выводит:

- число уникальных локаций;
- число уже сохранённых записей;
- exact и fuzzy результаты;
- ручные записи;
- отсутствующие или отклонённые локации.

Команда завершится с ошибкой, если покрытие меньше 100%.

## ✅ Покрытие и ошибки

Отсутствующая координата не считается допустимым состоянием.

- неизвестные локации обязательно логируются;
- невалидные координаты отклоняются;
- неоднозначные результаты не сохраняются;
- неполное покрытие блокирует публикацию нового кеша;
- страница артиста не выводит фиктивные `0, 0`;
- ошибка координат возвращает серверное состояние ошибки;
- существующий согласованный кеш не заменяется неполным снимком.

При необходимости координату можно добавить в JSON вручную с `source: "manual"`, но текущий справочник полностью заполнен автоматически.

## 🧪 Тестирование

Обычные offline-тесты:

```bash
go test ./...
go vet ./...
```

Тесты покрывают:

- нормализацию location keys;
- exact и fuzzy matching;
- жёсткую проверку страны;
- пустые, ошибочные и неоднозначные ответы Nominatim;
- timeout и HTTP errors через `httptest.Server`;
- rate limiting, включая fallback;
- чтение, проверку и безопасную запись JSON;
- полное покрытие локального fixture;
- координатный контракт SSR-шаблона;
- сортировку концертных локаций и дат;
- equirectangular projection;
- auto-fit, resize и antimeridian.

Live-проверка актуального Groupie Trackers API вынесена отдельно:

```bash
go test -tags=integration ./...
```

Она не запускается в обычном unit test suite.
