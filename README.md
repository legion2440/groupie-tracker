## 🚀 Как запустить

1. Склонировать и запустить проект
```bash
git clone https://01.tomorrow-school.ai/git/nyestaye/groupie-tracker
cd groupie-tracker
go run ./cmd/groupie-tracker
```
2. Когда проект запущен, в терминале вы увидите:
```bash
2025/06/20 17:45:21 Server started at http://localhost:8080
```
3. Перейдите на сайт http://localhost:8080

## 📝 Описание

Главная страница выводит карточки артистов / групп, детальная – полные **данные** по артисту / группе.
- данные `/artists`, `/locations`, `/dates`, `/relation` подгружаются кеш-воркером.
- названия городов приводятся к "Santiago, Chile"
- даты сортируются по времени
- Карта `locations → [dates]` формирует блок «Концерты по локациям»

**Система событий (event/action)**
- Кнопка «Обновить инфоримацию» отправляет `POST /api/refresh`; сервер асинхронно перезаполняет кеш.

**Асинхронность и кеш** 
- Фоновая горутина обновляет все данные эндпойнтов каждые 30 минут (TTL задаётся `CACHE_TTL`).
- Асинхронность обеспечивают **горутины** (4 параллельных HTTP-запроса в `refresh()`) и **каналы**, которые собирают ошибки без mutex-ов.

**HTTP-методы** 
- Только GET/POST; graceful-shutdown ловит `SIGINT/SIGTERM`.

**Проверка HTTP 500 (Internal server error)**  
- Отключите интернет → нажмите «Обновить информацию»;
- Сервер вернёт **HTTP 500**, кнопка станет красной («Ошибка обновления»);
- Удалением шаблонов вызвать 500 нельзя: все *.html встроены в бинарник через `//go:embed`.

**Шаблон ошибок** 
- `renderError` отдаёт `error.html` c нужным кодом.

**Unit-тесты**
- `api_test.go` - валидация парсинга внешнего API; 
- `handlers_test.go` - контент-type и статус код `/`;
- `visual_test.go` - статичные CSS/шаблоны (basic smoke-test).  

### Прочие "фишки"

**Cap-fallback:** 
- города преобразуются `london-uk` → `London, UK`; страны-аббревиатуры (`UK`, `USA`, `UAE`) выводятся правильным регистром.

**Сортировка дат:** 
- звёздочка (`*`) убирается, даты парсятся и сортируются по времени.

**«Полное имя» для сольного артиста:** 
- если `members` длиной 1 – вместо списка выводится `<b>Полное имя</b>: ...`.

**Embedded templates & static:** 
- HTML-шаблоны и JS встроены в бинарник (`//go:embed`), поэтому приложение работает из любого каталога без копирования ресурсов.  

**Кастомный TTL кеша:** 
- достаточно `export CACHE_TTL=10` - воркер будет обновляться каждые 10 минут.

**Image proxy cache**  
- `/img/{filename}.jpg` проксирует оригинальные обложки и отдаёт `Cache-Control: public, max-age=604800`;
- Браузер хранит файл 7 дней → повторный F5 не дёргает внешний API.

## 🧪 Тесты:
В проекте реализованы отдельные тестовые файлы для ключевых аспектов:

### core/api_test.go:
- проверка корректного парсинга и валидации внешнего API (endpoint /relation);
- убеждаемся, что парсер возвращает валидные структуры и ID.

### server/handlers_test.go:
Проверки на статус-коды (200/400/500), content-type и фирменные страницы ошибок:
- главная страница возвращает 200 и текстовый контент;
- тест негативных кейсов:
   - 500 (Internal Server Error) — по кастомному route;
   - 400 (Bad Request) — при неправильном методе на /api/refresh, 
      - что возвращается именно кастомная страница ошибки, а не дефолт Go.

### server/visual_test.go:
- проверка отдачи статики (CSS);
- проверка корректного отображения страницы 404, 
   - для несуществующего маршрута (/no/such/page) - возвращается 404 и фирменный шаблон ошибки.

**Запуск тестов**
```bash
go test ./...
```
или для подробного лога:

```bash
go test -v ./...
```
