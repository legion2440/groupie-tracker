# Groupie tracker

Веб-приложение на Go для просмотра артистов и групп из публичного Groupie Trackers API. Проект использует серверный рендеринг, согласованный кеш данных, поиск, фильтры, реальные концертные координаты и интерактивную Tour Map.

## 🚀 Как запустить

Требуется Go `1.24.2` или новее.

```bash
git clone https://01.tomorrow-school.ai/git/nyestaye/groupie-tracker
cd groupie-tracker
go run ./cmd/groupie-tracker
```

После запуска открыть:

```text
http://localhost:8080
```

По умолчанию кеш обновляется каждые 30 минут. Другой интервал можно задать через `CACHE_TTL`:

```bash
CACHE_TTL=10 go run ./cmd/groupie-tracker
```

## 📝 Описание

Приложение загружает данные из endpoint-ов `/artists`, `/locations`, `/dates` и `/relation`.

- четыре набора данных обновляются параллельно горутинами;
- новый кеш публикуется только после успешной загрузки полного согласованного снимка;
- главная страница выводит каталог артистов;
- страница артиста показывает состав, год создания, первый альбом и концерты;
- для сольного артиста выводится `FULL NAME`, для группы - `LINEUP`;
- даты концертов очищаются, проверяются и сортируются;
- локации отображаются в читаемом формате;
- аббревиатуры `UK`, `USA`, `UAE` сохраняют правильный регистр;
- страницы артистов имеют URL вида `/pink-floyd`;
- адреса из API `/artist/{id}` перенаправляются на canonical slug;
- ручное обновление выполняется через `POST /api/refresh`;
- ошибки `400`, `404` и `500` имеют отдельные состояния интерфейса;
- сервер корректно завершается по `SIGINT` и `SIGTERM`.

## ✨ Дополнительные возможности

- поиск и фильтры работают совместно без полной перезагрузки страницы;
- Back / Forward восстанавливают URL, результаты и элементы управления;
- светлая и тёмная темы сохраняются в `localStorage`;
- короткие аудиопревью ищутся через API Deezer канонические имена исполнителей проверяются через API MusicBrainz;
- одновременно воспроизводится только одно превью;
- изображения проходят через локальный proxy с HTTP-кешированием;
- HTML-шаблоны и статика встроены в бинарник через `go:embed`;
- frontend реализован на vanilla JavaScript без фреймворков.

### Обновление данных

Кнопка обновления отправляет `POST /api/refresh`. Сервер синхронно получает свежие данные и заменяет кеш только при успешном завершении всех запросов.
- если внешний API недоступен при доступности закешированных данных кнопка обновления загорается красным;
- если внешний API недоступен и инормация обновилась, кнопка загоряется зелёным на 5с и возвращается в исходное состояние;
- во время обновления кнопка крутится и горит жёлтым.

Если внешний API недоступен при первом запуске сервера:
- сервер возвращает фирменную страницу `500`;
- существующий согласованный кеш не заменяется неполными данными;
- интерфейс показывает состояние ошибки обновления.


### HTTP и ошибки

- приложение использует только необходимые `GET` и `POST` маршруты;
- ошибки 400 и 500 отображаются через общий шаблон error.html;
- для 404 используется отдельная фирменная страница 404.html;
- 404-страница обрабатывает неизвестные маршруты, отсутствующих артистов, отсутствующую встроенную статику и недоступные изображения;
- `SIGINT` и `SIGTERM` запускают graceful shutdown с таймаутом 5 секунд.


## ⚙️ Архитектура

### Backend

- стандартная библиотека Go для HTTP-сервера и маршрутизации;
- SSR через `html/template`;
- согласованный кеш на `atomic.Value`;
- отдельный каталог для нормализации, поиска, подсказок и фильтров;
- интеграции с Groupie Trackers API, Nominatim, MusicBrainz и Deezer;
- постоянный JSON-справочник географических координат.

### Frontend

- семантический HTML и CSS custom properties;
- небольшие независимые JavaScript-модули;
- частичное обновление SSR-разметки через `fetch` и `DOMParser`;
- History API для синхронизации URL;
- SVG и CSS для карты, маршрутов и анимаций;
- адаптивная компоновка для desktop, tablet и mobile.
