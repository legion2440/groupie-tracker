# Groupie tracker

Веб-приложение на Go для просмотра артистов и групп из публичного Groupie Trackers API. Проект объединяет серверный кеш, полнотекстовый и fuzzy-поиск, динамические фильтры, интерактивную визуализацию туров и аудиопревью треков.

## 🚀 Как запустить

Требуется Go `1.24.2` или новее.

1. Склонировать репозиторий и запустить проект:

```bash
git clone https://github.com/legion2440/groupie-tracker.git
cd groupie-tracker
go run ./cmd/groupie-tracker
```

2. После запуска в терминале появится адрес сервера:

```text
server started at http://localhost:8080
```

3. Открыть в браузере [http://localhost:8080](http://localhost:8080).

По умолчанию кеш обновляется каждые 30 минут. Интервал задаётся в минутах через переменную окружения:

```bash
CACHE_TTL=10 go run ./cmd/groupie-tracker
```

## 📝 Описание

Главная страница выводит карточки артистов и групп, а персональная страница — состав, год создания, дату первого альбома и концерты по локациям.

- данные `/artists`, `/locations`, `/dates` и `/relation` загружаются из публичного API;
- четыре endpoint-а обновляются параллельно отдельными горутинами;
- новый кеш публикуется только после успешной загрузки полного набора данных;
- каталог для поиска и фильтров строится из согласованного снимка `artists + relations`;
- даты концертов очищаются от `*`, парсятся и сортируются по времени;
- названия локаций приводятся к читаемому виду: `santiago-chile` → `Santiago, Chile`;
- аббревиатуры `UK`, `USA`, `UAE` сохраняют правильный регистр;
- для сольного артиста выводится `FULL NAME`, для группы — `LINEUP`;
- страницы артистов имеют человекочитаемые URL, например `/pink-floyd`;
- старые адреса `/artist/{id}` перенаправляются на канонический slug с кодом `301`.

### Обновление данных

Кнопка обновления отправляет `POST /api/refresh`. Сервер синхронно получает свежие данные и заменяет кеш только при успешном завершении всех запросов.

Если внешний API недоступен:

- сервер возвращает фирменную страницу `500`;
- существующий согласованный кеш не заменяется неполными данными;
- интерфейс показывает состояние ошибки обновления.

### HTTP и ошибки

- приложение использует только необходимые `GET` и `POST` маршруты;
- неизвестные маршруты и отсутствующая встроенная статика получают фирменную страницу `404`;
- некорректные параметры поиска или фильтров возвращают `400`;
- `SIGINT` и `SIGTERM` запускают graceful shutdown с таймаутом 5 секунд.

## 🔎 Поиск и фильтры

### Поиск

Поиск работает по:

- названию артиста или группы;
- участникам;
- концертным локациям;
- дате первого альбома;
- году создания.

Сначала сервер ищет точную фразу или подстроку. Если совпадений нет, включается поиск по токенам, затем — fuzzy fallback на основе ограниченного расстояния Левенштейна.

Fuzzy-поиск:

- допускает опечатки в именах, участниках и локациях;
- работает с Unicode-символами;
- использует адаптивный порог от 0 до 3 правок в зависимости от длины запроса;
- завершает вычисление раньше, если допустимая дистанция уже превышена;
- не размывает результаты, если найдено обычное точное совпадение.

Подсказки появляются с debounce `175 ms`, отменяют устаревшие запросы через `AbortController` и ранжируются по схеме: точное совпадение → начало строки → подстрока → токены → fuzzy.

Подробности: [Search bar](search-bar/README.md).

### Фильтры

Доступны:

- диапазон года создания;
- диапазон года первого альбома;
- количество участников, включая вариант `8+`;
- одна или несколько концертных локаций;
- локальный поиск по длинному списку городов.

Диапазоны лет и список локаций вычисляются из текущего каталога и повторно загружаются после обновления данных. Шкала количества участников фиксирована как `1–7` и `8+`, чтобы URL и интерфейс имели стабильную семантику.

Поиск и фильтры работают совместно. Изменяется только контейнер с карточками — без полной перезагрузки страницы. URL остаётся источником состояния, поэтому:

- параметры можно сохранить или отправить ссылкой;
- кнопки Back / Forward восстанавливают карточки и элементы управления;
- очистка поиска не сбрасывает выбранные фильтры;
- устаревший запрос отменяется, чтобы более медленный ответ не перезаписал новый.

Подробности: [Filters](filters/README.md).

## 🎨 Визуализация и UX

- светлая и тёмная темы с сохранением выбора в `localStorage`;
- системная тема применяется до загрузки CSS, поэтому нет заметного мигания интерфейса;
- локальные variable-font файлы Inter и Montserrat;
- адаптивная сетка карточек и фиксированная ширина панели фильтров на desktop;
- виниловые карточки поднимаются и начинают вращаться при наведении;
- большой винил на странице артиста также вращается при наведении;
- декоративная `TOUR MAP` синхронизирована со списком концертных локаций;
- SVG-маршрут строится как кривая Безье, а SVG-самолёт поворачивается по касательной к траектории;
- активная точка карты пульсирует, маршрут анимируется, геометрия пересчитывается через `ResizeObserver`;
- карточки локаций доступны с клавиатуры через `Enter` и `Space`;
- поддерживается `prefers-reduced-motion` для карты и favicon.

Все отдельные графические элементы анимаций - favicon, карта, маршрут, самолёт и интерфейсные иконки - выполнены в SVG. GIF, video и canvas не используются. Вращение виниловых DOM-элементов выполняется CSS-трансформацией, поэтому изображения остаются чёткими при любом размере.

### Анимированный favicon

Favicon представляет собой SVG-пластинку. Скрипт поворачивает группу `vinyl`, сериализует очередной SVG-кадр в `data:image/svg+xml` и обновляет `<link rel="icon">`. При включённом `prefers-reduced-motion` или ошибке загрузки остаётся статическая SVG-версия.

### Deezer API track preview

При наведении на карточку или большую пластинку приложение запрашивает у Deezer короткое превью трека выбранного артиста.

> Если браузер блокирует autoplay, после открытия страницы нужно один раз кликнуть в любом месте. После этого аудиопревью запускается при наведении.

- одновременно играет только одно превью;
- при уходе курсора воспроизведение останавливается и сбрасывается в начало;
- результаты Deezer кешируются на сервере и в браузере;
- таймаут внешнего запроса - 15 секунд;
- отсутствие трека или ошибка Deezer не ломают карточки и страницу артиста.

Подробности: [Visualizations](visualizations/README.md).

## ⚙️ Архитектура

### Backend

- HTTP-сервер и маршрутизация на стандартной библиотеке Go;
- кеш на `atomic.Value` с публикацией целого снимка;
- четыре параллельных запроса при обновлении API;
- отдельный нормализованный каталог для поиска, подсказок и фильтров;
- серверный рендеринг HTML-шаблонов;
- JSON endpoint-ы для подсказок, параметров фильтров и Deezer preview;
- image proxy `/img/{filename}` с `Cache-Control: public, max-age=604800`;
- HTML-шаблоны и статика встроены в бинарник через `//go:embed`.

### Frontend

- vanilla JavaScript без frontend-фреймворков;
- CSS custom properties для тем и дизайн-токенов;
- частичное обновление SSR-разметки через `fetch`, `DOMParser` и `replaceChildren`;
- History API для синхронизации URL и навигации Back / Forward;
- `AbortController` и request token-ы против гонок асинхронных запросов;
- SVG-графика без GIF, video и canvas.
